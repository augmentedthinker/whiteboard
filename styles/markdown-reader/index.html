Style 005

Markdown Reader

This is now better understood as a family of markdown-reading modes, not one fixed page type. Same markdown source, different degrees of interpretation — from raw source text to a fuller artifact wrapper.

Use this style when the source material is fundamentally markdown, but you want control over how literally or how aesthetically it should be presented.

What this style is actually becoming

Christopher’s critique was right: “Markdown Reader” was too broad to mean only one thing. So Style 005 now works more like a chooser between four markdown-viewer variants:

  • Raw Markdown — literal source text
  • Basic Rendered — minimal markdown rendering
  • Styled Reader — practical reading surface with modest chrome
  • Artifact Wrapper — markdown used inside a more authored page

  • “Use Style 005 · Basic Rendered”
  • “Use Style 005 · Styled Reader”
  • “Use Style 005 · Artifact Wrapper”
  • “Use Style 005 in mobile-reading view”

Minimal technical recipe

Markdown Reader (Style 005) is now a family, not one fixed surface.

Variant 1: Raw Markdown
- show the .md source directly in monospace

Variant 2: Basic Rendered
- markdown to HTML
- minimal spacing + basic typography

Variant 3: Styled Reader
- markdown to HTML
- restrained chrome + document frame + optional side rail

Variant 4: Artifact Wrapper
- markdown to HTML
- stronger hero/framing + more authored layout

Optional mobile-reading view
- reduce borders and empty space
- remove side rails / secondary blocks
- widen text to phone edges
- justify paragraphs for screen-filling readability
