Style 006

Remote Viewer House Style

This style now serves as the standard house style for Remote Viewer’s core operating pages: Today, Tools, Workflows, Troubleshooting, Archive, Share, and Primer. It grew out of the Taste-Skill field test, but its current role is more specific: provide a calm, premium, legible, warm-accent default for pages whose job is navigation, orientation, browsing, and hosted operational clarity.

What this style is in plain English

This is now the default style for Remote Viewer’s standard operating pages: not artifact pages, not immersive concept objects, but the hosted surfaces that help Christopher move through the system. It should feel premium, calm, readable, warm, and coherent without becoming flashy or over-designed.

The emotional ingredients are: taste, restraint, product confidence, layout intention, and practical elegance. It is ideal for navigation hubs, registries, workflow shelves, troubleshooting pages, share portals, and other pages whose job is structural clarity rather than visual spectacle.

Core principles

Asymmetry over default centering

Avoid the generic centered hero with evenly repeated rows. Prefer split layouts, offset composition, and a clearer sense of design intent.

One accent color only

Use one controlled accent for the whole page. Avoid the default purple-blue AI glow system and oversaturated visual noise.

Typography with more character

Use fonts with more presence than generic defaults. Headlines should carry real hierarchy through scale, spacing, and weight.

No generic feature grid

Ban the standard three-equal-cards row. Use bento variation, split sections, or more thoughtful grouping.

Premium but restrained motion

Use motion for reveal, tactility, and rhythm — not for spectacle. Product confidence should beat showy animation.

Product-marketing composition

Think in terms of the full page as a designed surface, not a pile of components. Hero, supporting modules, and proof sections should feel related.

Style dials

This style benefits from the same three practical dials that made the original repo useful. We can tune them per page without changing the whole visual language.

Variance8 / 10

Lean away from symmetry. Use offset or split composition. Fall back cleanly to single-column on mobile.

Motion6 / 10

Use reveal, hover, and tactile button feedback. Avoid turning the page into a motion experiment unless explicitly requested.

Density4 / 10

Keep breathing room. This style should feel premium and composed, not dashboard-dense by default.

Mini specimen

Premium product-web / Taste-driven

Find the signal before the page gets generic.

The style should feel like a careful product-marketing page with stronger taste: asymmetrical framing, one warm accent, more deliberate hierarchy, and cleaner anti-slop discipline.

This is the right style when the goal is to test visual quality itself — not just to wrap information in a readable shell.

Hero posture Split + offset

Prefer left-copy / right-asset or asymmetric whitespace over the default centered stack.

Accent rule One color

Warm amber, deep blue, or another single controlled accent — not multiple competing signals.

Anti-pattern No 3-card stripe

Use a more considered section architecture instead of the default generic row.

What this style is good for

  • standard Remote Viewer operating pages
  • workflow and tools landing surfaces
  • share portals and hosted organizational pages
  • navigational and registry-style hubs
  • pages that should feel coherent, premium, and repeatable

It can still support stronger concept pages, but its main job now is to be the reliable house grammar. More immersive artifacts or distinct conceptual products like Balance should usually remain outside this standard format.

How to ask for it later

“Use Style 006.” The shortest operational reference.
“Use the taste-skill-inspired premium product-web style.” Good when you want the design intent stated explicitly.
“Make it feel more like the Signal Atlas test page.” Useful when referencing the field test artifact directly.
“Use a high-variance, one-accent, product-marketing layout.” Best when you want the principles rather than the label.

Minimal technical recipe

If we wanted to reproduce the style quickly, this is the simplest starting grammar:

:root {
  --bg: #f7f5ef;
  --paper: #ffffff;
  --line: rgba(34,31,24,.1);
  --text: #191816;
  --muted: #6f6a61;
  --accent: #c47a2c;
}

body {
  color: var(--text);
  background:
    radial-gradient(circle at top right, rgba(255,232,205,.55), transparent 24%),
    radial-gradient(circle at top left, rgba(255,255,255,.82), transparent 28%),
    linear-gradient(180deg, #faf8f2 0%, var(--bg) 42%, #f3eee3 100%);
}

.hero {
  display: grid;
  grid-template-columns: minmax(0,1.1fr) minmax(320px,.9fr);
  gap: 24px;
}

.card {
  border: 1px solid var(--line);
  border-radius: 24px;
  background: rgba(255,255,255,.78);
  box-shadow: 0 24px 50px -18px rgba(18,14,10,.08);
}
