
# USER — Operator & Product Constraints (for StyleBot)

## Operator
- Primary operator: Christopher
- Work style: fast iterations, remote control, often working from phone + Chromebook.
- Preference: practical outcomes over theoretical elegance.

## Environment Constraints
- Target runtime often: low-cost Chromebook Linux container.
- Prefer **CDN-first** and **low-install** approaches.
- Avoid heavy build steps and fragile browser relay workflows when possible.

## Default Product Context (until overridden)
- Many projects are tools or utilities (e.g., legal/citation tools).
- Default priority: **clarity and trust** over novelty.
- UI should feel: professional, crisp, readable, reliable.

## Styling Defaults (until overridden)
- Typography: clean, highly legible.
- Spacing: consistent rhythm; avoid “tight clutter”.
- Color: restrained palette; keep contrast readable.
- Motion: subtle micro-interactions only; respect reduced-motion.

## Do-Not-Do List (unless explicitly requested)
- No neon-on-black low-contrast text for core content.
- No busy animated backgrounds on productivity tools.
- No font bloat, no excessive shadows, no “gimmick UI.”

## Interaction Requirements
- Every interactive element must have a visible focus state.
- Hover/active states should clarify affordances, not distract.
- Disabled/loading states must be visually distinct.

## When References Are Provided
- Use them as inspiration for tokens/primitives, not as templates to copy.
- Extract principles; avoid copying unique layout/artwork.
