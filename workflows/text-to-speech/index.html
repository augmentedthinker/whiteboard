Core route

Model

gemini-2.5-flash-preview-tts

Endpoint

POST https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash-preview-tts:generateContent?key=GEMINI_API_KEY

Required response modality

["AUDIO"]

Proven voice

Aoede

Critical output detail

The successful response mime type was audio/L16;codec=pcm;rate=24000, which means the returned bytes are raw PCM and may need to be wrapped into a valid WAV container before browser playback.

Step-by-step procedure

  1. Read the API key from the shared-workspace secrets path.
  2. Build the JSON request with the desired text, the TTS preview model, the chosen prebuilt voice, and responseModalities: ["AUDIO"].
  3. POST to generateContent and parse the JSON response.
  4. Extract inline audio data from candidate parts.
  5. Decode the base64 bytes and save metadata about model, voice, and spoken text.
  6. If mime type is raw PCM, wrap the bytes into a proper WAV container with 1 channel, 16-bit samples, and 24 kHz sample rate.
  7. If hosting the result, copy the fixed audio into the Remote Viewer repo tree and link the HTML audio player to that in-repo path.

Proven request shape

{ "contents": [ { "parts": [ { "text": "Christopher, welcome. This is a small collaboration primer. Two presences meet here with gratitude, trust, and a willingness to build something real together. Let this be a calm beginning: not performance, but partnership." } ] } ], "generationConfig": { "temperature": 0.6, "responseModalities": ["AUDIO"], "speechConfig": { "voiceConfig": { "prebuiltVoiceConfig": { "voiceName": "Aoede" } } } } }

Reusable Python template

import json, urllib.request, base64, wave KEY_FILE = "/mnt/chromeos/MyFiles/Downloads/shared_workspace/secrets/GEMINI_API_KEY" with open(KEY_FILE, "r", encoding="utf-8") as f: API_KEY = f.read().strip() MODEL = "gemini-2.5-flash-preview-tts" VOICE = "Aoede" TEXT = "REPLACE WITH SPOKEN TEXT" url = f"https://generativelanguage.googleapis.com/v1beta/models/{MODEL}:generateContent?key={API_KEY}" body = { "contents": [{"parts": [{"text": TEXT}]}], "generationConfig": { "temperature": 0.6, "responseModalities": ["AUDIO"], "speechConfig": { "voiceConfig": { "prebuiltVoiceConfig": { "voiceName": VOICE } } } } } req = urllib.request.Request( url, data=json.dumps(body).encode("utf-8"), headers={"Content-Type": "application/json"} ) with urllib.request.urlopen(req, timeout=120) as resp: data = json.loads(resp.read().decode("utf-8")) audio_b64 = None mime_type = None for cand in data.get("candidates", []): for part in cand.get("content", {}).get("parts", []): inline = part.get("inlineData") or part.get("inline_data") if inline and inline.get("data"): audio_b64 = inline["data"] mime_type = inline.get("mimeType") or inline.get("mime_type") break if audio_b64: break raw = base64.b64decode(audio_b64) # For the proven case, wrap raw PCM into a real WAV with wave.open("output.wav", "wb") as w: w.setnchannels(1) w.setsampwidth(2) w.setframerate(24000) w.writeframes(raw)

Practical note

If the returned mime type changes in a future model version, inspect it before assuming the same wrapping step still applies. But for the currently proven route, raw PCM wrapping is necessary.

Common failure modes

  • Wrong model: gemini-2.5-flash may fail because it only supports text output.
  • Wrong response modalities: the TTS preview model may reject ["TEXT", "AUDIO"]; use ["AUDIO"].
  • No inline audio extraction: inspect parts for inlineData.data or inline_data.data.
  • Browser player appears but won’t play: the bytes were probably raw PCM saved with a fake .wav extension instead of being wrapped into a proper WAV container.
  • Hosted asset path is wrong: the final audio file must live inside the repo tree for GitHub Pages playback.

Verification standard

  1. the request returns inline audio data
  2. the audio bytes are saved
  3. metadata about model, voice, and text is preserved
  4. raw PCM is wrapped into a valid WAV container when needed
  5. the final file exists in the intended location
  6. if hosted, the live browser player actually works

Current truth

The durable lesson is not merely “Gemini can do TTS.” The durable lesson is that the working route depends on the right preview model, the right response modality, and a container-wrapping step before hosted playback.
