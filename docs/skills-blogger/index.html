Important practical note

The access token itself is short-lived, but the refresh token lets the connection continue beyond the first hour. Google currently returned refresh_token_expires_in = 604799, which is about 7 days. So this specific testing-mode setup should be treated as a roughly one-week authorization window unless refreshed again or moved to a more stable production setup later.

Recommended usage going forward

  • Use Blogger for long-form, slower, review-first writing
  • Create drafts first; do not auto-publish without explicit request
  • If auth breaks again, re-auth should now be easier because the Cloud project and OAuth client are known and documented
