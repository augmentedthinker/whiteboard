✒️ Signature

Artifact: Image Generation Checker · 2026-03-25

Intent: Create a visible browser-side diagnostic surface for Gemini / Google image generation so image-model availability, auth, and actual image return behavior can be tested directly instead of guessed.

Initiating request

Christopher proposed adding an image generation checker related to the Tools lane, then explicitly asked to create it in Today’s Artifacts.

Direction rationale

  • Made it browser-side and client-only so it can use a manually pasted Gemini API key without depending on current Telegram-session credential exposure.
  • Included several likely image-capable Gemini/Google model IDs so success/failure can be compared quickly.
  • Shows real output previews when inline image data is returned.

Model / asset notes

  • Main authoring and implementation done directly in the OpenClaw main session.
  • No server-side secret handling; optional API key stays in browser localStorage only if the user saves it.
  • No generated images bundled with the tool itself.

Files

  • index.html — live artifact/tool surface
  • signature.html — provenance and implementation notes
